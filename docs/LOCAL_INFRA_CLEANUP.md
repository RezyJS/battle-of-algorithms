# Как потом снести локальную Docker/Colima-инфраструктуру

Этот файл нужен на будущее. Сейчас ничего удалять не надо.

Ниже безопасный порядок действий, если после завершения проекта ты захочешь убрать с ноутбука:

- `docker` CLI
- `docker-compose`
- `colima`
- локальные контейнеры, volumes и их данные

## Что у тебя может быть установлено

В нашем текущем сценарии используются:

- `docker` — CLI-клиент
- `docker-compose` — запуск `compose`-конфигов
- `colima` — локальный Docker daemon / VM на macOS

## Важное правило перед удалением

Сначала надо остановить проектные контейнеры и удалить их данные, и только потом сносить `colima` и brew-пакеты.

Иначе можно получить мусорные остатки и непонятное состояние.

## Вариант 1. Аккуратно удалить только инфраструктуру этого проекта

Если ты хочешь убрать только локальные контейнеры и данные именно этого проекта, но оставить `docker` и `colima` для других задач:

```bash
cd /Users/rezyjs/Development/battle-of-algorithms
docker-compose -f infra/docker-compose.yml down -v
```

Что делает команда:

- останавливает контейнеры проекта;
- удаляет контейнеры;
- удаляет volumes, созданные этим compose-проектом.

После этого можно проверить:

```bash
docker ps -a
docker volume ls
```

## Вариант 2. Полностью снести инфраструктуру проекта и сам локальный Docker runtime

### Шаг 1. Остановить и удалить проектные контейнеры

```bash
cd /Users/rezyjs/Development/battle-of-algorithms
docker-compose -f infra/docker-compose.yml down -v
```

### Шаг 2. Остановить Colima

```bash
colima stop
```

### Шаг 3. Если хочешь удалить все данные Colima VM

```bash
colima delete
```

Что это делает:

- удаляет виртуальную машину Colima;
- удаляет внутреннее состояние Docker daemon;
- удаляет локальные образы и контейнеры внутри этой VM.

Если хочешь просто остановить Colima и оставить данные на потом, делай только `colima stop`, без `colima delete`.

### Шаг 4. Удалить пакеты, установленные через Homebrew

```bash
brew uninstall colima
brew uninstall docker
brew uninstall docker-compose
```

Если какого-то пакета нет, Homebrew просто скажет об этом.

## Вариант 3. Полная очистка остатков после удаления

Это уже более жёсткий вариант. Используй только если точно хочешь всё подчистить.

Проверить наличие каталогов:

```bash
ls -la ~/.colima
ls -la ~/.docker
```

Если они остались и ты точно больше не хочешь сохранять настройки и данные:

```bash
rm -rf ~/.colima
rm -rf ~/.docker
```

Это удалит:

- конфиги Docker CLI;
- контексты Docker;
- состояние Colima;
- возможные локальные credentials и настройки.

## Как понять, что всё действительно удалено

Проверки:

```bash
which docker
which docker-compose
which colima
```

Если утилиты удалены, команды либо ничего не вернут, либо покажут ошибку вида `command not found`.

Проверка Docker:

```bash
docker info
```

После полного удаления команда не должна работать.

Проверка Colima:

```bash
colima status
```

После удаления команда тоже не должна работать.

## Рекомендуемый порядок удаления

Если потом решишь всё снести полностью, делай именно так:

1. `docker-compose -f infra/docker-compose.yml down -v`
2. `colima stop`
3. `colima delete`
4. `brew uninstall colima`
5. `brew uninstall docker`
6. `brew uninstall docker-compose`
7. при необходимости удалить `~/.colima` и `~/.docker`

## Что я рекомендую на практике

Если не уверен, нужен ли тебе Docker позже, не удаляй сразу пакеты.

Достаточно:

```bash
docker-compose -f infra/docker-compose.yml down -v
colima stop
```

Так ты освободишь ресурсы, но при этом всё можно будет быстро поднять обратно без новой установки.

## Когда точно не стоит удалять

Не сноси это, если ты:

- продолжаешь работать над этим проектом;
- хочешь локально тестировать `Postgres` и `Keycloak`;
- работаешь с другими проектами, где есть `docker-compose`;
- используешь локальные контейнеры для backend-разработки.
