# Битва алгоритмов — Документация проекта

## 1. Общее описание

**«Битва алгоритмов»** — веб-платформа для соревнования пользовательских алгоритмов навигации в лабиринте. Два оператора (управляемых скриптами на JavaScript) одновременно перемещаются по карте, собирают ключи и стремятся первыми добраться до выхода.

Платформа предназначена для образовательных целей: изучение алгоритмов поиска пути, оценки сложности и сравнения стратегий навигации.

## 3. Архитектура (Feature-Sliced Design)

Проект организован по методологии **FSD** — многослойная архитектура с чётким разделением ответственности.

```
src/
├── app/                         # Прикладной слой
│   └── model/
│       ├── game-store.ts        # Zustand store — центр управления боем
│       └── map-editor-store.ts  # Zustand store конструктора карт
│
├── shared/                      # Общий слой (переиспользуемое)
│   ├── model/
│   │   ├── enums.ts             # DirectionX, DirectionY, FieldContent
│   │   ├── types.ts             # StateSnapshot, FieldGrid, GridPoint
│   │   └── index.ts             # Public API
│   ├── lib/
│   │   └── utils.ts             # cn() — утилита для classnames
│   └── ui/
│       └── Navbar.tsx            # Навигационная панель
│
├── entities/                    # Бизнес-сущности
│   ├── operator/
│   │   ├── model/
│   │   │   ├── point.ts         # Класс Point — координаты с инкапсуляцией
│   │   │   └── operator.ts      # Класс Operator — основная игровая сущность
│   │   └── index.ts
│   └── field/
│       ├── model/
│       │   ├── static-field.ts  # Захардкоженная карта по умолчанию
│       │   ├── map-generator.ts # Генератор случайных проходимых лабиринтов
│       │   ├── field-validation.ts # Общая проверка достижимости BFS
│       │   └── field-utils.ts   # mergeFields — наложение операторов на карту
│       └── index.ts
│
├── features/                    # Пользовательские сценарии
│   ├── script-editor/
│   │   ├── model/
│   │   │   └── templates.ts     # 6 шаблонов алгоритмов
│   │   ├── ui/
│   │   │   └── ScriptEditor.tsx # Редактор кода (CodeMirror)
│   │   └── index.ts
│   └── battle-runner/

    ├── game-board/
    │   ├── ui/
    │   │   ├── GameBoard.tsx    # Игровое поле
    │   │   └── Cell.tsx         # Отдельная клетка

    │   │   └── ToolPalette.tsx   # Палитра кистей

        ├── ui/
```

### Правило зависимостей (FSD)

Слои зависят только от нижестоящих:

```
app → widgets → features → entities → shared
```

Запрещено: `shared` не может импортировать из `entities`, `entities` не может из `features`, и т.д.

## 4. Страницы приложения

### `/` — Арена

Главная страница для **просмотра боя**. Содержит:

- Игровое поле (визуализация лабиринта с операторами)
- Панель управления (старт/пауза, сброс, прогресс-бар, скорость)
- Переключение режимов `Гонка / Дуэль`
- Переключатель случайных карт с кнопкой перегенерации
- Переход в конструктор карт из режима дуэли
- Журнал событий (подбор ключа, выход)
- Легенда обозначений

### `/editor` — Редактор скриптов

Страница **программирования** алгоритмов. Содержит:

- Табы для переключения между игроками (каждый видит только свой код)
- CodeMirror-редактор с подсветкой JavaScript
- Выбор из 6 шаблонов (dropdown)
- Сворачиваемый справочник API оператора (аккордеон по категориям)
- Советы по написанию алгоритмов с примерами кода

### `/map-editor` — Конструктор карт

Страница ручной сборки арены. Содержит:

- Изменяемую сетку с drag-paint
- Палитру инструментов: стены, пустота, ключи, выход, спавны
- Настройку размеров в пределах лимитов генератора
- Валидацию карты перед применением
- Применение карты в текущую игру через `setCustomMap()`

### `/rules` — Правила

Описание правил соревнования:

- Цель игры
- Режимы `Гонка / Дуэль`
- Условия победы и таймаута по текущей игровой логике
- Система очков
- Правила взаимодействия операторов
- Ограничения (макс. шагов, размеры карты, язык, запрещённые API)
- Плавающее навигационное меню с подсветкой активной секции

## 5. Управление состоянием (Zustand)

Центральный store (`game-store.ts`) хранит:

| Поле | Тип | Описание |
|---|---|---|
| `currentStep` | `number` | Текущий шаг воспроизведения |
| `isRunning` | `boolean` | Идёт ли воспроизведение |
| `messages` | `string[]` | Лог событий |
| `histories` | `StateSnapshot[][]` | История состояний каждого оператора |
| `operators` | `Operator[]` | Экземпляры операторов |
| `scripts` | `string[]` | Код алгоритмов игроков |
| `field` | `FieldGrid` | Текущее состояние поля для рендеринга |
| `currentMap` | `FieldGrid` | Активная карта (статическая или сгенерированная) |
| `useRandomMap` | `boolean` | Флаг использования случайной карты |
| `gameMode` | `'race' \| 'duel'` | Режим боя |
| `mapWidth / mapHeight` | `number` | Текущий размер карты |
| `result` | `GameResult` | Итог боя и начисленные очки |
| `isGenerating` | `boolean` | Идёт ли анимированная генерация |

### Почему Zustand, а не React Context

- **Селективная подписка**: компоненты подписываются только на нужные поля, избегая лишних ре-рендеров (критично при обновлении `currentStep` каждые 200мс)
- **Нет Provider-обёртки**: состояние доступно из любого компонента без вложенности
- **Минимальный boilerplate**: в сравнении с Redux/RTK — значительно меньше шаблонного кода
- **Размер**: ~1.1 КБ (gzip)

## 6. Ключевые классы

### Point

Инкапсулированная позиция на карте. Приватные поля `#x`, `#y` защищены от прямой модификации. Предоставляет методы `getX()`, `getY()`, `changeX(direction)`, `changeY(direction)`, `clone()`.

### Operator

Основная сущность игрока. Хранит:

- Позицию (Point)
- Копию карты (для обзора)
- Флаги: `hasKey`, `hasExited`
- Историю состояний (для пошагового воспроизведения)
- Набор помеченных клеток (для алгоритмов с хлебными крошками)
- Счётчик шагов

**API оператора** (доступно из пользовательского скрипта):

| Метод | Категория | Описание |
|---|---|---|
| `moveRight/Left/Up/Down()` | Движение | Переместить оператора |
| `wait()` | Движение | Пропустить ход |
| `lookRight/Left/Up/Down()` | Обзор | Проверить проходимость |
| `scan(radius)` | Обзор | 2D-массив окружающих клеток |
| `getDistanceTo(target)` | Навигация | Манхэттенское расстояние до цели |
| `getPosition()` | Навигация | Текущая позиция |
| `getOpponentPosition()` | Навигация | Позиция соперника в режиме `duel` |
| `markCell()` | Метки | Пометить текущую клетку |
| `isMarked(dx, dy)` | Метки | Проверить метку со смещением |
| `hasKey()` | Состояние | Ключ подобран? |
| `hasExited()` | Состояние | Оператор вышел? |
| `getStepCount()` | Состояние | Количество шагов |

## 7. Исполнение пользовательского кода

Файл `execute-script.ts` принимает скрипт пользователя и выполняет его через `new Function('operator', scriptString)`.

**Меры безопасности:**

- Обёртка над методами оператора считает шаги и бросает ошибку при превышении лимита (1000)
- Пользовательский код не имеет доступа к DOM, сети, файловой системе — только к объекту `operator`
- Ошибки перехватываются и отображаются в интерфейсе

## 8. Правила соревнования

### Режимы

- **Гонка** — карта может быть стандартной или случайно сгенерированной; алгоритм не должен знать её заранее
- **Дуэль** — карта фиксирована; дополнительно доступен `getOpponentPosition()`

### Условия победы

1. **Ранний выход** — если оба оператора вышли, побеждает тот, кто сделал это раньше по шагам
2. **Единственный финишировавший** — если выйти успел только один, он побеждает
3. **Одновременный выход** — если оба вышли на одном шаге, фиксируется ничья
4. **Таймаут** — если никто не вышел, выигрывает оператор, у которого есть ключ; если ключ есть у обоих или ни у кого, это ничья

### Очки

| Действие | Очки |
|---|---|
| Ключ есть в финальном состоянии | +50 |
| Оператор вышел из лабиринта | +100 |
| Бонус победителю за ранний выход | +25 |
| Таймаут без выхода | 0 |

### Взаимодействие

- Операторы **блокируют** друг друга (не могут занять одну клетку)
- **Нет прямого боя** — соревнование на эффективность навигации
- Карта может быть статической, случайной или собранной в конструкторе
- Для кастомной карты обязательны 2 спавна, 1 выход, 1–2 ключа и достижимость целей от обоих игроков

## 9. Генерация карт

Алгоритм генерации описан подробно в файле `docs/ALGORITHMS.md`.

Ключевое свойство: **любая сгенерированная карта гарантированно проходима** — общая функция `validateReachability()` верифицирует достижимость всех целей от обоих спавнов. Та же проверка используется и в конструкторе карт.
